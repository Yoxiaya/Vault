账号密码管理APP（React Native版）技术栈文档

本文档基于账号密码管理APP需求文档，明确采用React Native跨平台开发方案的详细技术栈选型，涵盖客户端、加密、同步、交互等核心模块，为开发、测试提供统一技术依据，兼顾开发效率、双端兼容性与数据安全性。

一、核心技术框架（客户端）

- 核心框架：React Native 0.72+，一套代码适配iOS、Android双端，减少重复开发，提升开发效率，同时保证双端交互体验一致性。

- UI组件库：选用React Native Paper或NativeBase（二选一），组件丰富、适配性强，支持自定义样式，贴合APP简约易用的界面需求，降低UI开发成本。

- 导航管理：采用React Navigation 6.x，实现页面间流畅跳转，支持栈导航、底部Tab导航（对应APP首页、分类、密码生成、我的四大入口），适配双端导航逻辑。

- 状态管理：使用Redux Toolkit或MobX，统一管理全局状态（如用户登录状态、账号列表数据、分类信息、同步状态等），确保组件间数据同步，提升代码可维护性。

二、安全与加密技术（核心重点）

- 核心加密算法：采用AES-256对称加密算法，对所有账号密码进行加密处理，配合SHA-256哈希算法处理主密码摘要，防止密码明文泄露。

- 本地敏感存储：使用react-native-keychain插件，安全存储加密密钥、主密码摘要等敏感信息，避免敏感数据被非法获取，适配双端原生安全存储能力。

- 本地数据库加密：选用Realm JS或Watermelon DB（加密版），作为本地账号密码存储数据库，支持全量加密，确保本地数据即使被获取也无法破解。

- 生物识别集成：集成react-native-local-auth插件，实现指纹解锁、面容识别功能，统一双端调用逻辑，适配APP登录验证需求，提升安全性与便捷性。

三、网络与云端同步技术

- 网络请求：采用Axios库，封装统一网络请求拦截器，添加身份验证、数据加密处理，确保请求参数安全，同时处理请求失败、超时等异常场景。

- 传输加密：所有网络传输（含云端同步、备份）采用HTTPS（TLS 1.3）加密，防止数据在传输过程中被截取、篡改。

- 云端服务：选用阿里云ECS服务器，搭配阿里云OSS存储备份数据，阿里云RDS MySQL数据库存储用户基础信息（不含明文密码），保障云端数据稳定安全。

- 实时同步：通过WebSocket实现多设备实时同步，搭配增量同步机制，仅同步变更数据，减少带宽占用，提升同步速度；离线状态下暂存本地操作，网络恢复后自动同步。

四、核心功能相关插件与工具

- 系统级自动填充：集成react-native-autofill插件，iOS端适配ASAutofillCredentialProvider，Android端适配AutofillService，实现第三方APP、浏览器的密码自动填充功能。

- 轻量数据存储：使用react-native-mmkv插件，存储APP配置、用户偏好等轻量数据，高性能、低占用，提升APP运行流畅度。

- 异常监控：集成react-native-firebase/crashlytics或Sentry，实时监控APP崩溃、异常行为，收集报错信息，便于开发排查修复。

五、开发与构建工具

- 包管理：使用npm或yarn，统一管理项目依赖，确保开发环境一致性。

- 构建工具：Android端使用Android Studio，iOS端使用Xcode，配合React Native官方构建流程，实现双端打包、测试、上线。

- 多环境配置：集成react-native-config，支持开发、测试、生产多环境切换，避免环境配置冲突，提升开发效率。

六、适配与兼容性要求

- 系统版本：支持iOS 12.0及以上、Android 8.0及以上版本，适配主流手机品牌（华为、小米、苹果、OPPO、vivo等）。

- 双端一致性：确保UI界面、交互逻辑、功能实现双端统一，无明显差异；适配不同屏幕尺寸，避免拉伸、错位现象。

- 性能适配：优化组件渲染效率，减少卡顿、闪退，确保APP启动速度、操作响应速度符合需求文档中的性能指标。
